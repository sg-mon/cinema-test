# Cinema test

## Запуск

- Скопируйте содержимое `.env.example` в `.env`
- Запустите `npm i && npm run dev`
- - Если будут проблемы с запуском, попробуйте удалить `package-lock.json` и заново повторить предыдущий шаг
- - Если все равно будт, можно попробовать запустить с `yarn`

- MacOs
- node v20.19.5
- npm v10.9.0

## Комментарии

- `~/shared/ui` в nuxt 3 ломается при сборке из-за версии нитро. Так что перенесла в `~/components/ui`

- В апи не хватает пагинации для сущностей

- В букинге приходят `movieSessionId` и `sessionId`. Видимо `sessionId` забыли удалить из сваггера, поле нигде не используется

- С отображением билетов была проблема:

Для этого кейса фронту нужно поле `data` в букинге со свободными данными.

С текущим апи варианты следующие.

Загружать сначала данные по сеансу, потом список фильмов и кинотеатров и так много раз.

Либо пытаться загрузить как можно больше сеансов, идя по списку фильмов. Мерджить результаты, фильтруя на уникальность.

Оба варианта неудачные. Здесь ошибка апи, так что сделала симуляцию этого поля в локальном хранилище.
